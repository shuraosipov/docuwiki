=====Системные требования служб удаленной установки=====

Данный раздел описывает системные требования, предъявляемые к серверам служб удаленной установки RIS (Remote Installation Services), компьютерам-клиентам и сетевым адаптерам.

==Требования к службам сети==
  * В сети имеется сервер DNS и что сеть построена на основе протокола TCP/IP. Не обязательно использовать версию DNS, произведенную корпорацией Майкрософт.
  * В сети имеется авторизованный сервер DHCP. Не обязательно использовать версию протокола DHCP, произведенную корпорацией Майкрософт. Версия протокола DHCP от другого изготовителя не обязательно должна быть авторизованной.
  * Наличие Active Directory в сети.

==Требования к оборудованию серверов==

  * Оборудование сервера должно отвечать минимальным требованиям устанавливаемого продукта семейства Windows Server 2003. Например, если устанавливается Windows Server 2003, Enterprise Edition, компьютер должен отвечать минимальным требованиям этого продукта. Дополнительные сведения см.в разделе Windows Server 2003, Enterprise Edition. Системные требования.
  * Диск емкостью не менее 4 гигабайт для дерева папок сервера RIS.
  * Сетевой адаптер, поддерживающий протокол TCP/IP, совместимый с Windows, со скоростью 10 или 100 Мбит/сек (рекомендуется 100 Мбит/сек). 
  * Сервер RIS не может быть многосетевым компьютером, на нем может быть установлен только один сетевой адаптер.

==Требования к оборудованию компьютеров-клиентов==

  * Оборудование компьютера-клиента должно отвечать минимальным требованиям устанавливаемой операционной системы.
  * Загрузочное ПЗУ PXE по протоколу DHCP версии 1.00 или более поздней.

==Поддерживаемые сетевые адаптеры клиентов==
Чтобы получить список сетевых адаптеров, поддерживаемых RIS, запустите служебную программу Rbfg.exe. Дополнительные сведения см. в разделе Создание диска удаленной загрузки.

==Прочие вопросы==

  * RIS требуют значительного дискового пространства. Для дерева папок RIS следует выделить целый раздел диска.
  * RIS не могут быть установлены на один диск с системным или загрузочным томом.
  * Том, на который устанавливаются RIS, должен быть отформатирован с использованием файловой системы NTFS.
  * На данный момент RIS не поддерживают шифрованную файловую систему EFS (Encrypting File System). 
  * Распределенная файловая система DFS (Distributed File System) не поддерживается в качестве конечного расположения, однако службы RIS могут быть запущены на компьютере, на котором запущена DFS.
  * Каталог Microsoft Windows содержит перечень систем и оборудования, которые прошли тщательную проверку на совместимость и стабильность работы в продуктах семейства Windows Server 2003. Каталог Microsoft Windows является источником сведений для служб технической поддержки Майкрософт о том, поддерживается ли использование конкретной системы с продуктами семейства Windows Server 2003. Дополнительные сведения о совместимости оборудования см. в разделе Вспомогательные ресурсы.


Во время первого запуска мастера будет выведен запрос для указания расположения дерева каталогов \RemoteInstall. Это расположение должно находиться на томе с файловой системой NTFS и не должно находиться на системном или загрузочном томе.

=====Установка и настройка службы удаленной установки в Windows Server 2003 Standard=====
**Установка служб RIS с помощью мастера компонентов Windows.**

  * **Запуск установки службы удаленной установки** 
//Пуск - Панель инструментов - Установка и удаление программ - Установка компонентов Windows - Служба удаленной установки.//

>>Требуется установочный диск и права администратора, также потребуется перезагрузка.




**Настройке RIS.**

  * **Запуск мастера Установки служб удаленной установки**
//Пуск Администрирование - Установка служб удаленной установки//

>> Требуются права администратора.



  * **Мастер служб удаленной установки**
//Откроется окно Мастера служб удаленной установки, с описанием требований для установки. Жмем "Далее".//



  * **Местонахождение папки удаленной установки**
//Необходимо указать место, где создать папку удаленной установки. Вводим путь, жмем "Далее".//

>> Размещение папки удаленной установки должно соответствовать требованиям установки RIS.


  * **Исходные параметры**
//Настраиваем режим ответа на запросы клиентских компьютеров. Отмечаем галочки, жмем "Далее".//

>> Необходимо ответить пункт "Отвечать клиентским компьютерам, запрашивающим обслуживание".
>> Для повышения безопасности можно отметить пункт "Не отвечать неизвестным клиентским компьютерам".
 
  * **Местонахождение установочных файлов**:
//Указываем расположение инсталляционных файлов (диск или папка), жмем "Далее"// 

  * **Имя папки образа установки Windows**
//Указываем имя папки на сервере RIS, в которую будут скопированы инсталляционные файлы, жмем "Далее"//

>> Имя папки может быть любым. Например: PC-Kassa или Server

  * **Понятное описание и текст справки**
//Указываем описание установочного образа, которое будет отображаться при выборе дистрибутива для установки, жмем "Далее"//

>>Описание и текст лучше давать латиницей. Например: Dist-WXPPRo-PC-Kassa. В противном случае при установке описание и текст не будут видны!


  * **После проведения всех подготовительных процедур начнется процесс создания установочного образа. После завершения жмем "Готово."**

  * **Создание пользовательской учетной записи в Active Direcroy**

  * **Добавление Имени узла сервер загрузки в параметры области DHCP сервера**
//Пуск - DHCP - Параметры области - Настроить параметры - 066 Имя узла сервера загрузки - %Имя нашего сервера%//

>> Без добавления этой записи в параметры области DHCP сервер клиент не сможет получить список образов для установки.

  * **Установка разрешений на добавление компьютера в домен**
//Пуск-Active Directory Users And Computers-%Наш домен%-Делегирование-Добавляем группу "Все"-Присоединение компьютера к домену-Готово//

>>Без делегирования на добавление компьютеров в домен удаленная установке не получится

  * **Установка разрешений на образы и папку установки**
//Пуск-Active Directory Users And Computers-%Наш сервер RIS%-Свойства-Удаленная установка-Дополнительные параметры-Образы-Свойства-Разрешения-%Установить полный доступ для группы "Everyone" ("Все")%//

//Мой компьютер - "Диск с папкой установки RIS-Папка RIS-Свойства-Доступ-%Установиить полный доступ для группы "Everyone" ("Все")"
//

>> Без добавления разрешений клиентский компьютер не сможет увидеть установочный образы.
